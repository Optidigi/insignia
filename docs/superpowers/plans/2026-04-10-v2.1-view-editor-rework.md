# V2.1 View Editor Rework — Implementation Plan

> ## Execution Model
>
> **Main orchestrator:** Opus — reads this plan, dispatches subagents, reviews checkpoint results, makes judgment calls.
>
> **Implementation subagents:** Sonnet (`model: "sonnet"`) — executes each task. One fresh subagent per task. Each subagent receives: the task description, exact file paths, code to write, and verification commands. The subagent does NOT see the full plan — only its task.
>
> **Checkpoint subagents:** Sonnet (`model: "sonnet"`) — runs between phases. Receives a verification checklist and reports PASS/FAIL per item. Blocking gate — implementation does not proceed until all checks pass.
>
> **Final verification:** Opus (`model: "opus"`) — comprehensive 25-point audit after all phases complete. This is the last gate before the work is considered done.
>
> ## Rules
>
> - After each task, the implementation subagent MUST run `npm run typecheck && npm run lint` and confirm both pass before committing.
> - After any UI task, the subagent MUST take a Playwright screenshot (JPG format) and compare against the .pen design frame using `mcp__pencil__get_screenshot` on `admin-dashboard-v2.1-final.pen`.
> - If a checkpoint reports ANY failure, the orchestrator dispatches a fix subagent (Sonnet) for the specific issue, then re-runs the checkpoint.
> - Commits happen per-task, not per-phase. Each task produces one atomic commit.
>
> **Design reference**: `admin-dashboard-v2.1-final.pen` — use Pencil MCP `get_screenshot` to compare implementation against design frames.

**Goal:** Rework the View Editor right panel from the current tabbed/preset layout to a zone-centric panel with inline pricing, ruler calibration, overflow selectors, and improved storefront sizing UX. Also fix 3 adjacent issues: terminology sweep, duplicate scaleFactor bug, and toast+undo for deletions.

**Architecture:** Zone-centric right panel (no tabs). Each print area expands to show placement fee + logo sizes (always ≥1). Default size dropdown. Editable inline tier rows with name/scale/price. Clone modal with searchable setup list. Ruler tool stores calibration on ProductView. Storefront size step becomes card-based (2+ sizes) or preview step (1 size).

**Tech Stack:** React 18, React Router 7, Polaris v13, Prisma/PostgreSQL, Konva.js, Shopify App Bridge, Zod, TypeScript (strict)

- [ ] **Step 1: Build the `cloneLayoutInto` service function**

```typescript
/**
 * Clone print areas, positions, sizes, and pricing from a source config
 * INTO a target config. Replaces all existing placements on the target.
 */
export async function cloneLayoutInto(
  shopId: string,
  targetConfigId: string,
  sourceConfigId: string,
) {
  // Verify both configs belong to this shop
  const [source, target] = await Promise.all([
    getProductConfig(shopId, sourceConfigId),
    getProductConfig(shopId, targetConfigId),
  ]);
  if (!source || !target) throw new AppError(ErrorCodes.NOT_FOUND, "Config not found", 404);

  return db.$transaction(async (tx) => {
    // Delete all existing placements on target (cascade deletes steps)
    await tx.placementDefinition.deleteMany({
      where: { productConfigId: targetConfigId },
    });

    // Delete all existing view-level geometry on target views
    await tx.productView.updateMany({
      where: { productConfigId: targetConfigId },
      data: { placementGeometry: Prisma.DbNull },
    });

    // Copy placements from source
    for (const placement of source.placements) {
      const newPlacement = await tx.placementDefinition.create({
        data: {
          productConfigId: targetConfigId,
          name: placement.name,
          basePriceAdjustmentCents: placement.basePriceAdjustmentCents,
          hidePriceWhenZero: placement.hidePriceWhenZero,
          defaultStepIndex: placement.defaultStepIndex,
          displayOrder: placement.displayOrder,
        },
      });

      if (placement.steps.length > 0) {
        await tx.placementStep.createMany({
          data: placement.steps.map((step) => ({
            placementDefinitionId: newPlacement.id,
            label: step.label,
            scaleFactor: step.scaleFactor,
            priceAdjustmentCents: step.priceAdjustmentCents,
            displayOrder: step.displayOrder,
          })),
        });
      }
    }

    // Copy view-level geometry from source views to matching target views (by perspective)
    const sourceViews = await tx.productView.findMany({
      where: { productConfigId: sourceConfigId },
      select: { perspective: true, placementGeometry: true },
    });
    for (const sv of sourceViews) {
      if (sv.placementGeometry) {
        await tx.productView.updateMany({
          where: { productConfigId: targetConfigId, perspective: sv.perspective },
          data: { placementGeometry: sv.placementGeometry },
        });
      }
    }
  });
}
```

- [ ] **Step 2: Build the `CloneLayoutModal` component**

Create `app/components/CloneLayoutModal.tsx`:

```typescript
import { Modal, TextField, ResourceList, ResourceItem, Text, InlineStack, Banner, Button } from "@shopify/polaris";
import { SearchIcon } from "@shopify/polaris-icons";
import { useState, useMemo } from "react";

type SetupItem = {
  id: string;
  name: string;
  viewCount: number;
  placementCount: number;
  methodNames: string[];
};

type Props = {
  open: boolean;
  onClose: () => void;
  onApply: (sourceConfigId: string) => void;
  setups: SetupItem[];
  loading?: boolean;
};

export function CloneLayoutModal({ open, onClose, onApply, setups, loading }: Props) {
  const [search, setSearch] = useState("");
  const [selectedId, setSelectedId] = useState<string | null>(null);

  const filtered = useMemo(() => {
    if (!search) return setups;
    const q = search.toLowerCase();
    return setups.filter((s) => s.name.toLowerCase().includes(q));
  }, [setups, search]);

  return (
    <Modal
      open={open}
      onClose={onClose}
      title="Clone from another setup"
      primaryAction={{
        content: "Apply",
        disabled: !selectedId,
        loading,
        onAction: () => selectedId && onApply(selectedId),
      }}
      secondaryActions={[{ content: "Cancel", onAction: onClose }]}
    >
      <Modal.Section>
        <Text as="p" tone="subdued">
          Select a product setup to copy its print areas, positions, sizes, and pricing into this configuration.
        </Text>
      </Modal.Section>
      <Modal.Section>
        <TextField
          label="Search setups"
          labelHidden
          placeholder="Search setups..."
          value={search}
          onChange={setSearch}
          prefix={<Icon source={SearchIcon} />}
          autoComplete="off"
        />
        <div style={{ maxHeight: 240, overflow: "auto", marginTop: 8 }}>
          <ResourceList
            items={filtered}
            renderItem={(item) => (
              <ResourceItem
                id={item.id}
                onClick={() => setSelectedId(item.id)}
                selected={selectedId === item.id}
              >
                <Text as="span" fontWeight={selectedId === item.id ? "semibold" : "regular"}>
                  {item.name}
                </Text>
                <Text as="span" tone="subdued">
                  {" "}— {item.viewCount} views · {item.placementCount} print areas · {item.methodNames.join(", ")}
                </Text>
              </ResourceItem>
            )}
          />
        </div>
      </Modal.Section>
      <Modal.Section>
        <Banner tone="warning">
          This will replace all print areas, positions, sizes, and pricing in this setup.
        </Banner>
      </Modal.Section>
    </Modal>
  );
}
```

```typescript
const pixelDistance = Math.sqrt(
  Math.pow((pointB.x - pointA.x) * imageWidth, 2) +
  Math.pow((pointB.y - pointA.y) * imageHeight, 2)
);
const pxPerCm = unit === "in" 
  ? pixelDistance / (distance * 2.54)
  : pixelDistance / distance;
```
